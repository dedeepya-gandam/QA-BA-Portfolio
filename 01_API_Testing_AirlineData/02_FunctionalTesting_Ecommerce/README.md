# Functional & Regression Testing Project – E‑Commerce Web Application

This project showcases end‑to‑end functional and regression testing for a sample e‑commerce web
application. It is designed as a portfolio piece to demonstrate real‑world QA deliverables.

## Application Under Test (AUT)

**Name:** ShopSmart E‑Commerce Web App (fictitious but realistic)  
**Modules Covered:**
- User Registration & Login
- Product Search & Product Details
- Shopping Cart
- Checkout & Payment (simulated)
- Order History

## Test Objectives

- Verify that core user journeys work as expected:
  - New user registration and login
  - Searching and viewing products
  - Adding/removing items from the cart
  - Completing a checkout with valid data
- Perform negative and boundary testing on all critical input fields.
- Build a regression test suite that can be reused for future releases.
- Capture defects in a structured defect log and map test coverage using an RTM.

## Files in this folder

- `TestPlan_Ecommerce_WebApp.md` – High‑level test plan
- `TestScenarios_Ecommerce_WebApp.xlsx` – Scenario‑level breakdown
- `TestCases_Ecommerce_WebApp.xlsx` – Detailed test cases
- `DefectLog_Ecommerce_WebApp.xlsx` – Sample defect report/log
- `RTM_Ecommerce_WebApp.xlsx` – Requirements Traceability Matrix
- `screenshots/` – Sample UI & defect evidence screenshots
